# [ADP-351] Range Request

## Guidance

- SHOULD response with `Accept-Ranges` to inform the client that the server is supporting Range Request no matter the client request involve any Range Request related header.
    
    ```jsx
    HTTP/1.1 200 OK
    Accept-Ranges: bytes
    ```
    
- MAY send `Accept-Ranges: none` to tell the client we do not support Range Request for the requesting resource.
- SHOULD take `Range` header in the request and return the partial response if supported.
    - SHOULD return with HTTP status 206 Partial Content in this case.
    - SHOULD return with HTTP Header Content-Range in this case to indicate what ranges the returning response belongs to.
    - SHOULD return with HTTP Header Accept-Ranges in this case.
    
    ```jsx
    GET /resources/r-id HTTP/1.1
    Ranges: bytes=0-1023
    
    HTTP/1.1 206 Partial Content
    Content-Range: bytes 0-1023/146515
    Content-Length: 1024
    Accept-Ranges: bytes
    ```
    
    - The server SHOULD response HTTP status 416 if the requested Range is not satisfied or the unit is not supported. See HTTP Headers - Range.
