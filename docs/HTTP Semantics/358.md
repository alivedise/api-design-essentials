# [ADP-358] Partial Request

## Overview

The main purpose of filtering is to add more flexibility to a collection for the user interface to decide what specific data to fetch.

There’re two meaning of filtering

- Result filtering: filter the result by the condition(a=b, x>z)
- Field filtering: filter the result content by the given key only

## Guidance

- [SHOULD support filter](311.md)
- SHOULD use `filters` as payload for POST method since we recommend to use payload for POST
    
    Notice: To use the payload, it’s not easy to design for value comparison. SHOULD design a new property parallel to filters
    
    ```jsx
    POST /search
    {
      filters: {
        belongedCountry: 'Taiwan',
        shape: {
          size: 'SQUARE'
        }
      }
    }
    ```
    

## Example

```jsx

GET /articles?include=author&fields[articles]=title,body&fields[people]=name HTTP/1.1
GET /resource?filters%5Bstatus_id%5D=1&filters%5Bcity%5D=Sydney&page=2&include=relatedResource
```

### Basic Filtering with Comparison Operators in Query Parameters

**Endpoint**: `GET /products`

**Filters**:

- Category: `category`
- Price range: `price<`, `price<=`, `price>`, `price>=`
- Availability: `inStock`

**Example Request**:

```bash
GET /products?filter[category]=electronics&price<=500&price>=100&inStock=true
```

**Description**: This request retrieves electronic products priced between 100 and 500 units that are currently in stock.

### Advanced Filtering with Multiple Comparison Operators

**Endpoint**: `GET /users`

**Filters**:

- Age range: `age<`, `age<=`, `age>`, `age>=`
- Signup date: `signUpAt`
- Status: `status`

**Example Request**:

```bash
GET /users?filter[age]>=18&filter[age]<=30&filter[signUpAt]=2023-01-01&filter[status]=ACTIVE
```

**Description**: This request retrieves active users aged between 18 and 30 who signed up after January 1, 2023.

### Complex Filtering with Request Body and Comparison Operators

**Endpoint**: `POST /search`

**Request Body**:

```json
{
  "filters": {
    "category": "books",
    "price": {
      "gte": 10,
      "lte": 50
    },
    "author": "John Doe",
    "publishedAt": {
      "before": 
    } 
  }
}
```

**Description**: This request searches for books authored by John Doe, priced between 10 and 50 units, and published between January 1, 2020, and January 1, 2023.

### Filtering with Not Equal To Operator

**Endpoint**: `GET /orders`

**Filters**:

- Order status: `status!=`

**Example Request**:

```bash
GET /orders?status!=CANCELLED
```

**Description**: This request retrieves all orders that are not cancelled.
