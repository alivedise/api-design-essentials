# [ADP-357] Long running job

## Guidance

- SHOULD return Location in the response header to give the client the information of created job.
- SHOULD have a status field for the job resource
- SHOULD return HTTP Status 202 Accepted for a successful job creation
- MAY attach Sunset to indicate the job would be removed after completion.

```json
{
  "jobId": "uuid-",
  "status": "RUNNING",
}
```

- [DRAFT] SHOULD give problem (compatible with HTTP Problem JSON) or error json if the job failed
    
    ```jsx
    GET /jobs/{job-id} HTTP/1.1
    
    HTTP/1.1 200 OK
    
    {
      "jobId": "uuid",
      "status": "FAILED",
      "problem": {
        "type": "/problems/api-invoke-error",
        "title": "API invoke error"
      }
    }
    ```
