# [ADP-366] Idempotency

## Guidance

- SHOULD be idempotent for GET/PUT/DELETE in any case.
- SHOULD implement Idempotent for POST/PATCH.
    - See HTTP Headers - Idempotency-Key

## Design Thinking

- Microsoft raised another proposal to deal with Idempotency. See https://docs.oasis-open.org/odata/repeatable-requests/v1.0/repeatable-requests-v1.0.html
- The server implementation shall store the Idempotency-Key in a store such as database or cache to manage consistent API behavior.
