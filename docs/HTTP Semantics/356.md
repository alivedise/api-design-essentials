# [ADP-356] Batch Request

## Guidance

- SHOULD choose between two common ways to do batch request/response.
    - Use `multipart/mixed`
    - Have a batch action with HTTP Post

## Batch Request (HTTP `multipart/mixed`)

- SHOULD support the execution of multiple operations in a single HTTP request to improve efficiency and reduce network overhead.
- SHOULD maintain the same level of validation and error handling for each individual operation within the batch as if they were executed separately.
- MAY include different types of operations (e.g., GET, POST, PUT, DELETE) within a single batch request.
- SHOULD limit the number of operations per batch to prevent server overload and ensure manageable request sizes.
- SHOULD support both synchronous and asynchronous processing of batch requests.
- SHOULD include metadata about the batch request to enable clients to understand the context and status of the operations.

### Batch Request (POST `/resource/batch`)

- SHOULD support the execution of multiple operations in a single HTTP request to improve efficiency and reduce network overhead.
- SHOULD maintain the same level of validation and error handling for each individual operation within the batch as if they were executed separately.
- MAY include different types of operations (e.g., GET, POST, PUT, DELETE) within a single batch request.
- SHOULD limit the number of operations per batch to prevent server overload and ensure manageable request sizes.
- SHOULD support both synchronous and asynchronous processing of batch requests.
- SHOULD include metadata about the batch request to enable clients to understand the context and status of the operations.

### Batch Response

- SHOULD return a single HTTP response containing the results of all operations included in the batch request.
- SHOULD include individual status codes and header/body for each operation to provide detailed feedback on the success or failure of each.
    
    ```json
    {
      "batch": [
        {
          "status": 201,
          "header": {
            "Content-Type": "application/json"
          },
          "body": {
          }
        }
      ]
    }
    ```
    
- SHOULD maintain the order of operations in the response to match the order of operations in the request.
- SHOULD include error details for any failed operations to assist in troubleshooting and resolution.
- SHOULD NOT fail if one of the request failed.
- SHOULD return the status code for overall batch request.

### Example

### Batch Request Example (HTTP `multipart/mixed`)

```jsx
POST /batch HTTP/1.1
Content-Type: multipart/mixed; boundary=boundary

--boundary
Content-Type: application/http
Content-Transfer-Encoding: binary

POST /api/resources HTTP/1.1
Content-Type: application/json

{
  "name": "Resource 1",
  "type": "Type A"
}
--boundary
Content-Type: application/http
Content-Transfer-Encoding: binary

PUT /api/resources/123 HTTP/1.1
Content-Type: application/json

{
  "name": "Updated Resource"
}
--boundary
Content-Type: application/http
Content-Transfer-Encoding: binary

DELETE /api/resources/456 HTTP/1.1
--boundary--
```

### Batch Response Example (HTTP `multipart/mixed`)

```jsx
HTTP/1.1 200 OK
Content-Type: multipart/mixed; boundary=boundary

--boundary
Content-Type: application/http
Content-Transfer-Encoding: binary

HTTP/1.1 201 Created
Content-Type: application/json

{
  "id": "789",
  "name": "Resource 1",
  "type": "Type A"
}
--boundary
Content-Type: application/http
Content-Transfer-Encoding: binary

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": "123",
  "name": "Updated Resource"
}
--boundary
Content-Type: application/http
Content-Transfer-Encoding: binary

HTTP/1.1 204 No Content
--boundary--
```

### Batch Request Example (POST `/resource/batch`)

```jsx
POST /batch HTTP/1.1
Content-Type: application/json

{
  "batch": [
    {
      "method": "POST",
      "url": "/api/resources",
      "body": {
        "name": "Resource 1",
        "type": "Type A"
      }
    },
    {
      "method": "PUT",
      "url": "/api/resources/123",
      "body": {
        "name": "Updated Resource"
      }
    },
    {
      "method": "DELETE",
      "url": "/api/resources/456"
    }
  ]
}
```

### Batch Response Example (POST `/resource/batch`)

```jsx
HTTP/1.1 200 OK

{
  "batch": [
    {
      "status": 201,
      "body": {
        "id": "789",
        "name": "Resource 1",
        "type": "Type A"
      }
    },
    {
      "status": 200,
      "body": {
        "id": "123",
        "name": "Updated Resource"
      }
    },
    {
      "status": 204,
      "body": null
    }
  ]
}

```
