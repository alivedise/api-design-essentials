# [ADP-23] Developer Experience

> The sum of all interactions and events, both positive and negative, between a developer and a library, tool, or API.
> 

The software client(web, application) is the interface to the normal users; API is the `interface` of developers. 

- How do I use the API to complete my task?
- How could I use the library?
- How do I make this code re-usable by other developers?
    - Even you are in a small team with the only developer is yourself; the one after 3 month will ALWAYS forget why do you design this then.

## Design the Developer Experience

- naming - is the variable/function name/API describe itself clearly?
- consistence - is the pattern being found and applied everywhere in the same project?
- communication - is the idea being promoted without a bump in the road, even it is not that good?
