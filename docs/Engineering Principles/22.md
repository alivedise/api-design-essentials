# [ADP-22] API First

## Overview

![From github.com/ByteByteGoHq/system-design-101](%5BADP-22%5D%20MUST%20follow%20API%20First%209978aa328bf34d359a7308e2355c81fb/Untitled.png)

From github.com/ByteByteGoHq/system-design-101

API-first is essentially `contract-first`. The entire team collaborates from the outset to establish a contract. This allows frontend engineers, backend engineers, and test programmers to work independently on their implementations while adhering to the agreed-upon contract.

## Reference

- from https://www.youtube.com/watch?v=twfufQn1mD0
- An API is a promise. [https://blog.axway.com/learning-center/apis/basics/an-api-is-a-promise](https://blog.axway.com/learning-center/apis/basics/an-api-is-a-promise)
- An API is nice to be important, but more important to be nice. [https://x.com/rogerfederer/status/494599597193314304](https://x.com/rogerfederer/status/494599597193314304)
- APIs
    - shall be technically correct
        - keep the promise to the clients and never break the contract
    - should be respectful, courteous
        - make contract more pleasant
