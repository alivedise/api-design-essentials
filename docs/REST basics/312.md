# [ADP-312] Pagination

## Overview

**Pagination** in RESTful APIs is a technique used to split large sets of data into smaller, more manageable chunks or "pages." This helps improve performance, reduce load on servers, and provide a better user experience. There are two primary methods of pagination: offset-based and cursor-based.

### Offset-Based Pagination

Offset-based pagination uses an offset value to specify the starting point of the data set and a limit to specify the number of items to retrieve.

### Example

```jsx
GET /resources?offset=20&limit=10 HTTP/1.1
```

```jsx
{
  "resources": [ /* resources from offset 20 to 29 */ ],
  "meta": {
    "offset": 20,
    "limit": 10,
    "totalItems": 100
  },
  "links": {
    "self": "/resources?offset=20&limit=10",
    "next": "/resources?offset=30&limit=10",
    "prev": "/resources?offset=10&limit=10",
    "first": "/resources?offset=0&limit=10",
    "last": "/resources?offset=90&limit=10"
  }
}
```

### Cursor-Based Pagination

Cursor-based pagination uses a cursor (usually a unique identifier) to fetch the next set of results.

### Example

```jsx
GET /resources?cursor=abc123&limit=10 HTTP/1.1
```

```jsx
{
  "data": [ /* resources starting after cursor abc123 */ ],
  "meta": {
    "next_cursor": "def456",
    "limit": 10
  },
  "links": {
    "self": "/resources?cursor=abc123&limit=10",
    "next": "/resources?cursor=def456&limit=10"
  }
}
```

## Guidance

- SHOULD choose of the pagination convention and use the common query parameters for pagination. Currently cursor-based is more common.
- SHOULD provide next/prev page link in response header
    
    ```jsx
    Link: <https://api.example.com/resources?page=2>; rel="next",
          <https://api.example.com/resources?page=1>; rel="prev",
          <https://api.example.com/resources?page=1>; rel="first",
          <https://api.example.com/resources?page=10>; rel="last"
    ```
    

## Example

### OpenAPI sample

```jsx
openapi: 3.1.0
info:
  title: Sample API
  version: 1.0.0
paths:
  /resources:
    get:
      summary: Get a list of resources
      parameters:
        - name: page
          in: query
          description: Page number for pagination
          required: false
          schema:
            type: integer
        - name: filter
          in: query
          description: Filter criteria
          required: false
          schema:
            type: string
        - name: sort
          in: query
          description: Sort order
          required: false
          schema:
            type: string
      responses:
        '200':
          description: A paginated list of resources
          headers:
            Link:
              description: Pagination links
              schema:
                type: string
          content:
            application/json:
              schema:
                type: object
                properties:
                  resources:
                    type: array
                    items:
                      type: object
                  total:
                    type: integer
                  page:
                    type: integer
                  pageSize:
                    type: integer
```
