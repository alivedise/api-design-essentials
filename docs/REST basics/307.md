# [ADP-307] Inspiring URI Design Collection

Here is a list of good uri design when navigating the public API documents, as inspiring material for API designers.  Add more if you see a good API design.

## Guidance

- If the design violates certain ADP, append the violating ADP id.
- This document would focus on URN design, not the content or header.
- SHOULD specify why it’s inspiring
- SHOULD review the list and try to find out the pattern we could re-use in the ADP of URI design pattern.

## Pattern: Non pluralized path

## Github

### meta/get

```
Get GitHub meta information
GET/meta

Description
Returns meta information about GitHub, including a list of GitHub's IP addresses. For more information, see "About GitHub's IP addresses."
```

- The API endpoint is not a collection nor a “resource”.

### meta/get-all-versions

```
Get all API versions
GET/versions
Get all supported GitHub API versions.

Responses
[
  "2021-01-01T00:00:00.000Z",
  "2021-06-01T00:00:00.000Z",
  "2022-01-01T00:00:00.000Z"
]
```

- The API does not return a real collection of resources but an array of string.

### app-hook-config/get

```
GET/app/hook/config

Returns the webhook configuration for a GitHub App. For more information about configuring a webhook for your app, see "Creating a GitHub App."
```

- None of the path is plural

## Pattern: Modifier

### **apps/scope-token**

```
POST/applications/{client_id}/token/scoped

Use a non-scoped user-to-server access token to create a repository scoped and/or permission scoped user-to-server access token. You can specify which repositories the token can access and which permissions are granted to the token. You must use Basic Authentication when accessing this endpoint, using the client_id and client_secret of the GitHub App as the username and password. Invalid tokens will return 404 NOT FOUND.
```

### apps/list-accounts-for-plan

```
GET/marketplace_listing/plans/{plan_id}/accounts
Returns user and organization accounts associated with the specified plan, including free plans. For per-seat pricing, you see the list of accounts that have purchased the plan, including the number of seats purchased. When someone submits a plan change that won't be processed until the end of their billing cycle, you will also see the upcoming pending change.
```

## Pattern: MISC

### apps/get-by-slug

```
GET/apps/{app_slug}

Note: The :app_slug is just the URL-friendly name of your GitHub App. You can find this on the settings page for your GitHub App (e.g., https://github.com/settings/apps/:app_slug).
```

- The design specifies that identifier might not be a meaningless uuid but a url friendly string.
