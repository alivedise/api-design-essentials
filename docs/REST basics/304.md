# [ADP-304] Plural form for Collection

## Guidance

- SHOULD use plural form to represent `collection`, which means a collection of specific resource, in the URI.
    
    ```jsx
    GET /features HTTP/1.1
    
    HTTP/1.1 200 OK
    {
      features: [
        {
          name: 'WebRTC'
        }
      ]
    }
    ```
    
- SHOULD use `results`: as field name for search related API which does not be relevant to specific resources.
    
    ```jsx
    POST /search HTTP/1.1
    
    HTTP/1.1 200 OK
    {
      results: []
    }
    ```
    

## Design Thinking

This’d be one of the most common convention for REST api. On the other hand, we may need to put something which is not plural in certain case. For instance, singular form might be used for endpoints representing actions or commands rather than collections of resources.

## Example

DO

```jsx
GET /planets/earth HTTP/1.1
GET /api-guidelines HTTP/1.1
POST /apis HTTP/1.1
```

DON’T

```jsx
GET /api HTTP/1.1
```
