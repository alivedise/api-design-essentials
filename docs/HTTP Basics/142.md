# [ADP-142] X-Request-Id

## Overview

This document establishes the guidelines for utilizing the `X-Request-Id` header to track requests between clients and servers within our system. The use of `X-Request-Id` aims to enhance traceability, improve debugging processes, and ensure consistent tracking of requests across distributed services and microservices.

## Scope

This guideline applies to all client and server applications within our organization that communicate over HTTP/HTTPS. It is particularly relevant for services involved in processing multiple requests, including microservices architectures where requests might span multiple services.

## Guidance

### Definition

`X-Request-Id` is an HTTP header used to carry a unique identifier for each request. It’s not standard but conventional. This identifier allows tracking and correlating logs across different services and systems involved in processing the request.

### Requirements

- The client SHOULD generate a unique `X-Request-Id` for each outgoing request.
    - If the client does not generate a `X-Request-Id`, the first server that receives the request SHOULD generate a unique `X-Request-Id` and include it in the response and any subsequent requests.
- Any service, including microservices, that receives a request with an `X-Request-Id` SHOULD propagate this header with the same value to any downstream services or components involved in processing the request.
    - The `X-Request-Id` **SHOULD NOT** be altered once set for a particular request flow.
- All services, including microservices, SHOULD include the `X-Request-Id` in their logs. This allows for correlation of logs across different services, making it easier to trace the entire lifecycle of a request.
    - Monitoring and alerting systems SHOULD be configured to use the `X-Request-Id` for tracking and debugging purposes.
- The `X-Request-Id` SHOULD be a UUID (Universally Unique Identifier) to ensure uniqueness across all requests.
    - Example: `X-Request-Id: 123e4567-e89b-12d3-a456-426614174000`

### Best Practices

- When returning error responses, the server SHOULD include the `X-Request-Id` in the response headers. This allows clients to reference the request ID when reporting issues or seeking support.
