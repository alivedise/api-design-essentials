# [ADP-125] Content-Range

## Overview

The `Content-Range` header is a critical component in supporting partial content delivery in HTTP responses. It is used to specify which part of the resource is being returned in a partial response or to inform the client about the total size of the resource when a range request cannot be satisfied. This functionality is particularly useful in scenarios where large files need to be downloaded or streamed in segments, ensuring efficient use of bandwidth and providing better control over the data transfer process.

In designing APIs that support range requests, it is essential to adhere to the specifications outlined in RFC 9110. The `Content-Range` header should only be used with `206 Partial Content` responses, indicating a successful partial content delivery, or with `416 Range Not Satisfiable` responses, indicating that the requested range is invalid.

## Guidance

- SHOULD return Content-Range while the API is designed to support Range-Request with successful response.
    
    ```jsx
    Content-Range: bytes 0-499/1234
    Content-Range: bytes 500-999/1234
    Content-Range: bytes 500-1233/*
    ```
    
- SHOULD return Content-Range for a Range Request API but the request is not satisfied. (as RFC 9110 suggested)
    
    ```jsx
    HTTP/1.1 416 Range Not Satisfiable
    Content-Range: bytes */1234
    ```
    
- As RFC 9110 specified, the header should only work with 206 or 416 response.

## Example

### OpenAPI 3.1.0 Example

```yaml
openapi: 3.1.0
info:
  title: Range Request API
  version: 1.0.0
paths:
  /resource:
    get:
      summary: Get partial content
      parameters:
        - name: Range
          in: header
          required: true
          schema:
            type: string
            example: bytes=0-499
      responses:
        '206':
          description: Partial Content
          headers:
            Content-Range:
              description: Indicates the range of bytes returned
              schema:
                type: string
                example: bytes 0-499/1234
          content:
            application/octet-stream:
              schema:
                type: string
                format: binary
        '416':
          description: Range Not Satisfiable
          headers:
            Content-Range:
              description: Indicates the size of the full resource
              schema:
                type: string
                example: bytes */1234
          content:
            application/problem+json:
              schema:
                type: object
                properties:
                  type:
                    type: string
                    example: about:blank
                  title:
                    type: string
                    example: Range Not Satisfiable
                  status:
                    type: integer
                    example: 416
                  detail:
                    type: string
                    example: The range specified is not valid for the resource.
```
