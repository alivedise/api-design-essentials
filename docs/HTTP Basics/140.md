# [ADP-140] Sunset

## Overview

The `Sunset` HTTP Header is used to indicate that a resource will no longer be available after a specific date and time. This helps API clients better understand and manage the lifecycle of API resources during development and maintenance.

## Guidance

- API which `audience≥PARTNER_EXTERNAL` SHOULD set the `Sunset` header to support API lifecycle management. This helps to clearly communicate the expiration date of resources and ensures that clients can prepare appropriate actions. See SHOULD support API life cycle management.
- SHOULD set Sunset for temporary resource handling
    - A resource generated for temporary reference, and would be removed after being finished/deleted.
    
    ```jsx
    GET /jobs/{job-id} HTTP/1.1
    
    HTTP/1.1 200 OK
    Sunset: Sat, 6 Jul 2024 19:43:31 GMT
    
    {
      jobId: '....',
      state: 'SUCCESS'
    }
    ```
    
- SHOULD set Sunset for resource retention
    - A resource would be purged due to service limitation.
    
    ```jsx
    GET /reserved-seats HTTP/1.1
    
    HTTP/1.1 200 OK
    Sunset: Sat, 26 Oct 2024 19:43:31 GMT
    ```
    

## Example

### OpenAPI Example

Here is an example of how to document the use of the `Sunset` header in an OpenAPI 3.1.0 specification:

```yaml
openapi: 3.1.0
info:
  title: Example API
  version: 1.0.0
paths:
  /jobs/{job-id}:
    get:
      summary: Get job status (temporary resource)
      responses:
        '200':
          description: Successful response
          headers:
            Sunset:
              description: Indicates when the resource will be sunset
              schema:
                type: string
                format: date-time
                example: '2024-07-06T19:43:31Z'
          content:
            application/json:
              schema:
                type: object
                properties:
                  jobId:
                    type: string
                  state:
                    type: string
  /reserved-seats:
    get:
      summary: Get reserved seats (resource retention)
      responses:
        '200':
          description: Successful response
          headers:
            Sunset:
              description: Indicates when the resource will be sunset
              schema:
                type: string
                format: date-time
                example: '2024-10-26T19:43:31Z'
          content:
            application/json:
              schema:
                type: object
                properties:
                  seats:
                    type: array
                    items:
                      type: object
                      properties:
                        id:
                          type: string
                        status:
                          type: string
```
