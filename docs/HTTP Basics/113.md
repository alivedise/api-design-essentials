# [ADP-113] PUT

## Overview

The HTTP PUT method is used to update or replace a resource at a specified URI. It is an idempotent method, meaning that multiple identical requests should have the same effect as a single request. PUT is often used to update a resource with a new state.

## Guidance

- SHOULD ensure that multiple identical PUT requests have the same effect as a single request.
- MAY respond with the updated resource representation in the response body.
- SHOULD use the resource's unique identifier in the URI.
- MAY support using a complex key if the resource identification requires it.
- SHOULD replace the entire resource with the new state provided in the request body.
- MAY validate and reject the request if partial updates are attempted.
- SHOULD validate the incoming request for completeness and correctness.
- MAY return appropriate HTTP status codes (e.g., 400 Bad Request) if validation fails.
- **Responses**
    - SHOULD return a 200 OK status code if the update is successful and a representation of the updated resource is included.
    - SHOULD return 201 Created if the resource is newly created from the PUT request.
    - SHOULD return 202 Accepted if the request is about an asynchronous request and it succeeds.
    - SHOULD return a 204 No Content status code if the update is successful and no representation is included.
    - SHOULD return a 404 Not Found status code if the resource does not exist.
    - SHOULD return a 409 Conflict status code if there is a conflict in the update process.

### Example

1. **Update a User Profile**
    
    ```jsx
    PUT /users/123
    Content-Type: application/json
    
    {
      "name": "John Doe",
      "email": "john.doe@example.com",
      "age": 30
    }
    ```
    
    **Response**
    
    ```jsx
    HTTP/1.1 200 OK
    Content-Type: application/json
    
    {
      "id": 123,
      "name": "John Doe",
      "email": "john.doe@example.com",
      "age": 30
    }
    ```
    
2. **Replace an Existing Product**
    
    ```jsx
    PUT /products/456
    Content-Type: application/json
    
    {
      "name": "New Product Name",
      "price": 99.99,
      "stock": 50
    }
    ```
    
    **Response**
    
    ```jsx
    HTTP/1.1 204 No Content
    ```
    
3. **Resource Not Found**
    
    ```jsx
    PUT /products/789
    Content-Type: application/json
    
    {
      "name": "Non-Existent Product",
      "price": 0,
      "stock": 0
    }
    ```
    
    **Response**
    
    ```jsx
    HTTP/1.1 404 Not Found
    ```
