# [ADP-111] GET

## Overview

The HTTP GET method is used to request data from a specified resource. Requests using GET should only retrieve data and should not have any other effect on the data.

## Guidance

- **SHOULD be idempotent:** GET requests should not change the state of the resource. They can be repeated without causing different outcomes.
- **SHOULD be safe:** GET requests should not cause any side effects for which the user is held responsible.
- **SHOULD return cacheable responses:** Responses to GET requests should be cacheable to improve performance and reduce server load. See [Cache Control](361.md).
- **MAY support common query parameters:** Use query parameters to filter, sort, or paginate the data being requested. See [Pagination](312.md), [Sorting](313.md), [Filtering](314.md).

# Examples

### Example 1: Retrieve a list of users

**Request:**

```jsx
GET /users
```

**Response:**

```jsx
HTTP/1.1 200 OK

[
    {
        "id": 1,
        "name": "Alice"
    },
    {
        "id": 2,
        "name": "Bob"
    }
]
```

### Example 2: Retrieve a single user by ID

**Request:**

```jsx
GET /users/1 HTTP/1.1
```

**Response:**

```jsx
HTTP/1.1 200 OK

{
    "id": 1,
    "name": "Alice"
}

```
