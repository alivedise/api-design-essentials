# [ADP-114] DELETE

## Overview

The HTTP DELETE method is used to delete a specified resource. When a DELETE request is received, the server should delete the resource identified by the URI and return a response indicating the outcome of the operation.

## Guidance

- **SHOULD Be Idempotent**
    - The DELETE method should be idempotent, meaning that multiple identical DELETE requests should have the same effect as a single request. Sending multiple DELETE requests to the same resource should not cause additional side effects.
- Responses
    - **MAY Return 204 No Content**
        - Upon successful deletion, the server may return a `204 No Content` status code, indicating that the resource was successfully deleted and there is no additional content to send in the response body.
    - **MAY Return 202 Accepted**
        - If the deletion process is asynchronous or the resource cannot be immediately deleted, the server may return a `202 Accepted` status code, indicating that the request has been accepted for processing but the deletion has not been completed yet.
    - **SHOULD Return 404 Not Found for Non-existent Resources**
        - If the resource specified in the DELETE request does not exist, the server should return a `404 Not Found` status code.
- **MAY Use Conditional Requests**
    - The server may support conditional DELETE requests using headers such as `If-Match` to ensure that the resource is only deleted if it matches a specified condition, preventing accidental deletions due to outdated or conflicting data.

### Example

- **Successful Deletion**
    
    ```jsx
    DELETE /api/v1/resources/123 HTTP/1.1
    Host: example.com
    Authorization: Bearer <token>
    
    HTTP/1.1 204 No Content
    ```
    
- **Non-existent Resource**
    
    ```jsx
    DELETE /api/v1/resources/999 HTTP/1.1
    Host: example.com
    Authorization: Bearer <token>
    
    HTTP/1.1 404 Not Found
    ```
    
- **Unauthorized Request**
    
    ```jsx
    DELETE /api/v1/resources/123 HTTP/1.1
    Host: example.com
    
    HTTP/1.1 401 Unauthorized
    ```
    
- **Conditional Deletion**
    
    ```jsx
    DELETE /api/v1/resources/123 HTTP/1.1
    Host: example.com
    Authorization: Bearer <token>
    If-Match: "etag123"
    
    HTTP/1.1 204 No Content
    ```
    
- **Accepted for Processing**
    
    ```jsx
    DELETE /api/v1/resources/123 HTTP/1.1
    Host: example.com
    Authorization: Bearer <token>
    
    HTTP/1.1 202 Accepted
    ```
