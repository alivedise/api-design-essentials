# [ADP-117] Patch: Content-Type

# Guidelines for Choosing JSON Patch and Merge Patch

When designing and implementing an API, selecting the appropriate patch type (JSON Patch or Merge Patch) is crucial for ensuring the API's efficiency and maintainability. Below are detailed guidelines on when to use `application/json-patch+json` and `application/merge-patch+json`.

### JSON Patch (`application/json-patch+json`)

**Suitable Scenarios:**

- **Complex Operations:** When you need to perform multi-step or complex operations on a JSON document, such as adding, replacing, and removing multiple fields simultaneously.
- **Fine-Grained Control:** When precise manipulation of specific parts of a JSON document is required, such as an element in an array or a property in a nested object.
- **Diverse Operation Types:** When you need to use various operation types such as `add`, `remove`, `replace`, `move`, `copy`, and `test`.

**Example:**

```json
[
  { "op": "replace", "path": "/email", "value": "newemail@example.com" },
  { "op": "add", "path": "/phone", "value": "+1234567890" }
]
```

**Guidelines:**

- MUST specify `Content-Type: application/json-patch+json` in the request header.
- MUST validate the structure and validity of the patch document.
- SHOULD use JSON Patch for scenarios that require multiple operations or complex updates.
- SHOULD ensure the atomicity of patch operations, meaning all operations must either succeed or fail together.
- SHOULD clearly document the supported JSON Patch operation types and their usage.

### Merge Patch (`application/merge-patch+json`)

**Suitable Scenarios:**

- **Simple Partial Updates:** When only a few fields in a JSON document need to be updated without requiring complex structural changes.
- **Field Replacement:** When you simply need to replace the values of existing fields.
- **Low Complexity Operations:** When the update operations are straightforward and do not involve arrays or complex nested objects.

**Example:**

```json
{
  "email": "newemail@example.com",
  "phone": "+1234567890"
}
```

**Guidelines:**

- MUST specify `Content-Type: application/merge-patch+json` in the request header.
- MUST validate the merge patch document to ensure it is well-formed.
- SHOULD use Merge Patch for straightforward updates that do not require multiple operations or complex modifications.
- **SHOULD NOT** use Merge Patch if fine-grained control or multiple types of operations are needed.
- SHOULD document the usage of Merge Patch clearly, indicating when it is appropriate to use it.
