# [ADP-765] OpenAPI: timezone format

## Overview

OpenAPI schema does not cover timezone data as default format, so we introduced a new format from IANA tzdata.

## Guidance

- Must use customized `iana-timezone` data format (e.g., "America/New_York", "Europe/London").
- Must be represented as a string type.

```yaml
openapi: 3.1.0
info:
  title: Timezone API
  version: 1.0.0
paths:
  /example:
    get:
      summary: Example endpoint
      responses:
        '200':
          description: Successful response
          content:
            application/json:
              schema:
                type: object
                properties:
                  timezone:
                    type: string
                    format: iana-timezone
                    description: Timezone in IANA format
                    example: "America/New_York"
components:
  schemas:
    Timezone:
      type: string
      format: iana-timezone
      description: Timezone in IANA format
      example: "America/New_York"
```
