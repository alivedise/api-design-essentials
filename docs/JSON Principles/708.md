# [ADP-708] Timezone data

## Guidance

- SHOULD use IANA Time Zone Database for string which format tagged as timezone
All applications and services MUST use the IANA Time Zone Database for representing and handling time zone information. This requirement ensures that time zone data is accurate, up-to-date, and consistent across all systems.

## IANA Time Zone Database Overview

The IANA Time Zone Database, also known as the tz database or zoneinfo database, is a globally recognized standard for time zone information. It provides a comprehensive and detailed set of time zone rules and histories, including:

1. **Time Zone Names**: Standardized identifiers for time zones, such as `America/New_York`, `Europe/London`, and `Asia/Tokyo`.
2. **Offset Information**: Details on the offset from Coordinated Universal Time (UTC) for each time zone, including historical changes due to daylight saving time and other adjustments.
3. **Transition Rules**: Rules for transitions between standard time and daylight saving time, including start and end dates.

### Example

An example of an IANA time zone identifier is:

```
America/New_York
```

This represents the time zone for New York, USA, including its standard time and daylight saving time rules.

## Implementation Detail

### Data Storage

- Time zone information stored in databases and other storage systems MUST use the IANA time zone identifiers.
- Conversions and calculations involving time zones SHOULD utilize libraries and tools that support the IANA Time Zone Database.

### Libraries and Tools

- Developers SHOULD use libraries and tools that support the IANA Time Zone Database to handle time zone data.
- It is RECOMMENDED to regularly update these libraries to ensure they reflect the latest changes in the IANA Time Zone Database.
