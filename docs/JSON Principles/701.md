# [ADP-701] Property Design

## Guidance

- Properties MUST use camelCase.
- Properties SHOULD NOT use special characters such as `:` `[` `]` `.` `;` `%`.

## Design Thinking

There are two common conventions for JSON properties:

1. camelCase
2. snake_case

While snake_case is sometimes used to match query parameter conventions like `underline_snakecase`, it is not as intuitive. Therefore, we choose to use camelCase for JSON properties.

<aside>
💡 When properties need to be included in URLs, convert camelCase to kebab-case to comply with our path naming conventions.

</aside>

## Example

```json
{
  "userName": "johnDoe",
  "emailAddress": "john@example.com"
}
```

**Query Parameter Example:**

```jsx
GET /users?filter.userName=johnDoe&filter.emailAddress=john@example.com HTTP/1.1
```

**URL Path Example**

```jsx
GET /users HTTP/1.1

HTTP/1.1 200 OK

{
  users: [
    {
      registeredApplications: []
    }
  ]
}

```

Let’s say we have registeredApplications as property of the user schema; when we need to apply the property in uri it needs to be transformed into kebab-case.

```jsx
GET /registered-applications HTTP/1.1
```
