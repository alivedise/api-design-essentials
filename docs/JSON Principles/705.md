# [ADP-705] name field convention

## Guidance

- MAY prefix `{resourceType}` for the name field instead of a simple `name`.

## Design Thinking

The `name` field might be a common field to the resource object, prefixing with resourceType makes it clear about the field is the name of the resource or something else.

## Example

DO

```jsx
{
  principleName: 'my lord'
}
```

DON’T

```jsx
{
  name: 'my lord'
}
```
