# [ADP-704] id field convention

## Guidance

- MAY prefix `{resourceType}` for the id field instead of a simple `id`.

## Design Thinking

Since the `id` field is a MUST for every resource object, it can be confusing when more than one resource object is contained (by sub-resource). Prefixing with resourceType makes it clear to identify the type of object.

## Example

DO

```jsx
{
  userId: 'uuid-v4...'
}
```

DON’T

```jsx
{
  id: 'uuid-v4...'
}
```
