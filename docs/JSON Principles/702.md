# [ADP-702] Enumerable value convention

## Guidance

- MUST use UPPER_SNAKECASE for enumerable format.

## Design Thinking

Although the naming of “type” field seems to be enumerable always, but it shall be uri-reference for `problem+json`. See MUST be compatible with HTTP Problem.

## Example

DO

```jsx
{
  "role": "GROUP_ADMINISTRATOR"
}
```

DON’T

```jsx
{
  "role": "groupAdministrator"
}
```

SHOULDN’T

```jsx
{
   "role": 1
}
```

- If the value is not used to compute for something later, already return string based enumeration.

## OpenAPI Example

```yaml
openapi: 3.0.0
info:
  title: Role API
  version: 1.0.0
paths:
  /roles:
    get:
      summary: Get roles
      responses:
        '200':
          description: A list of roles
          content:
            application/json:
              schema:
                type: array
                items:
                  type: object
                  properties:
                    role:
                      type: string
                      enum:
                        - GROUP_ADMINISTRATOR
                        - USER
                        - MODERATOR
                      example: GROUP_ADMINISTRATOR
    post:
      summary: Create a role
      requestBody:
        content:
          application/json:
            schema:
              type: object
              properties:
                role:
                  type: string
                  enum:
                    - GROUP_ADMINISTRATOR
                    - USER
                    - MODERATOR
                  example: GROUP_ADMINISTRATOR
      responses:
        '201':
          description: Role created
```
