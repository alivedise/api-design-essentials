# [ADP-703] date-time and time field convention

## Guidance

- SHOULD affix `At` for date-time field favour simply `time` or `timestamp` ,
- MAY use common field to represent date or time related field containing: `time`, `date` , `timestamp` , `day`

## Design Thinking

By adding the `At`, it explicitly indicates that the field is for date-time format, not a normal string. Although you could check the documentation for the string format, a good API should describe itself semantically.

If we don’t know the semantic of the time, use common string including time/timestamp/date/day is accepted. For example, in cloudevents it defines a `time` field which does not have solid meaning(created or current).

For the value part see MUST use RFC3339 to present date-time and time format.

## Example

DO

```jsx
{
  startAt: '2023-10-12T08:00:00Z',
  endAt: '2023-10-12T10:00:00Z'
}

```

DON’T

```jsx
{
  start: '2023-10-12T08:00:00Z',
  end: '2023-10-12T10:00:00Z',
  occur: '2023-10-12T10:00:00Z'
}
```
