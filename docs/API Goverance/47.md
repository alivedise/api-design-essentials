# [ADP-47] API Authentication and Authorization

## Guidance

- Each API endpoint SHOULD be secured by OAuth2.0 or JWT based authentication**.**
    - To ensure secure access to API endpoints, implement OAuth2.0 or JWT (JSON Web Token) for authentication. These methods provide robust security mechanisms, supporting token-based access and enabling various grant types for different use cases.
    - **Example:**
        - **OAuth2.0**: Use the authorization code grant type to authenticate users.
            
            ```jsx
            POST /oauth/token HTTP/1.1
            Host: api.example.com
            Content-Type: application/x-www-form-urlencoded
            
            grant_type=authorization_code&code=AUTH_CODE&redirect_uri=REDIRECT_URI&client_id=CLIENT_ID&client_secret=CLIENT_SECRET
            ```
            
        - **JWT**: Validate the JWT token on each API request.
            
            ```jsx
            GET /protected/resource HTTP/1.1
            Host: api.example.com
            Authorization: Bearer <jwt_token>
            ```
            
- SHOULD NOT use basic authentication.
    - Basic authentication is less secure because it transmits user credentials in an easily decodable format. Avoid using it to protect API endpoints.
    - DON’T:
        
        ```jsx
        GET /protected/resource HTTP/1.1
        Host: api.example.com
        Authorization: Basic <base64_encoded_credentials>
        ```
        
- SHOULD build on top of HTTPS.
    - All API communications must be encrypted using HTTPS to protect data in transit from eavesdropping and man-in-the-middle attacks. Ensure that the SSL/TLS certificates are correctly configured and maintained.
    - Exception: intranet based device MAY not provide https.
    - **Example:**(The URL should start with `https://`.)
        
        ```jsx
        GET /protected/resource HTTP/1.1
        Host: api.example.com
        ```
        
- **MAY design role-based authorization control (RBAC) by applications.**
    - Implement role-based access control to manage user permissions effectively. Define roles and assign permissions based on the roles to ensure users only access the resources they are authorized to use.
    - **Example:**
        - Define roles and permissions:
            
            ```json
            {
              "roles": [
                {
                  "name": "admin",
                  "permissions": ["read", "write", "delete"]
                },
                {
                  "name": "user",
                  "permissions": ["read"]
                }
              ]
            }
            ```
            
        - Assign roles to users:
            
            ```json
            {
              "users": [
                {
                  "username": "alice",
                  "roles": ["admin"]
                },
                {
                  "username": "bob",
                  "roles": ["user"]
                }
              ]
            }
            ```
