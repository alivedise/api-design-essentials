# [ADP-42] API Lifecycle Management

## Guidance

- MUST tell the client an API is being deprecated if the API audience is greater than `COMPANY_INTERNAL` by using following headers: Deprecation and Sunset.
- An retiring API is at first stepping into Deprecation state and then Sunset state. And retire.
- It’s possible to have both headers at the same time.

### Deprecation

Indicate the requesting API is being deprecated and might be unavailable in the future.

```jsx
Deprecation: @1688169599
```

- SHOULD use structured field date from RFC draft. Structured field is defined in 3.3.7 of https://datatracker.ietf.org/doc/draft-ietf-httpbis-sfbis/
- SHOULD provide Link in response to give the client more info about the deprecation. See [HTTP Headers - Deprecation](127.md)`

### Sunset

Indicate the requesting API would be unavailable at the specified date-time.

```jsx
Sunset: Wed, 11 Nov 2026 11:11:11 GMT
```

- SHOULD mark `deprecated` in OpenAPI document.
- Notice: Sunset could be used for an temporal resource would be unavailable soon.
