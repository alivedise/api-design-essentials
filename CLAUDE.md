# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Overview

This repository is the API Design Essentials (ADE) documentation project. It serves dual purposes:

1. A VitePress-based documentation site hosting comprehensive API design guidelines
2. An npm package (`api-design-essentials`) that provides a Redocly ruleset for validating OpenAPI specifications

The documentation is multilingual (English and Traditional Chinese) and covers comprehensive API design standards including HTTP basics, REST principles, error design, event design, and OpenAPI conventions.

## Development Commands

### Documentation Development

```bash
# Start development server for documentation
pnpm docs:dev

# Build documentation for production
pnpm docs:build

# Preview production build
pnpm docs:preview
```

### Linting and Quality

```bash
# Lint markdown files
pnpm docs:lint
```

## Architecture

### Documentation Structure

Documentation files are organized in `docs/` with the following pattern:

- `docs/en/` - English documentation
- `docs/zh-tw/` - Traditional Chinese documentation

Each category has its own subdirectory (e.g., `HTTP Basics/`, `REST basics/`, `Error Design/`, etc.)

### Document Naming and Frontmatter

All ADE documents follow a strict naming convention:
- Files are named by their ADE number: `{id}.md` (e.g., `100.md`, `401.md`)
- Each document must have frontmatter with at least:
  ```yaml
  ---
  id: {number}
  title: {document title}
  ---
  ```
- Optional frontmatter fields:
  - `overview: true` - marks overview documents
  - `draft: true` - marks draft documents
  - `placeholder: true` - marks placeholder documents

### VitePress Configuration

VitePress configuration is split across multiple files in `docs/.vitepress/config/`:

- `index.mjs` - Main configuration with PWA and Mermaid plugins, URL rewrites, and locale setup
- `shared.js` - Shared configuration across locales
- `en.js` - English locale configuration and sidebar generation
- `zh-tw.js` - Traditional Chinese locale configuration

### Sidebar Generation

The sidebar is dynamically generated by scanning markdown files in `docs/{locale}/` directories. The `getSidebar()` function in locale config files:

1. Recursively scans directories for `.md` files
2. Parses frontmatter using `gray-matter`
3. Generates sidebar items based on file metadata
4. Auto-generates `list.md` with all documents sorted by ID

## Key Conventions

### ADE Document Links

When working with ADE documents:
- Reference other ADEs by ID number in links: `[ADE-302](302)` not by filename
- The frontmatter `id` field is the source of truth for document identification
- Cross-references between documents should use the numeric ID

### Adding New ADE Documents

1. Create the markdown file as `{id}.md` in the appropriate category folder under both `docs/en/` and `docs/zh-tw/`
2. Add frontmatter with `id` and `title` fields
3. The sidebar and list page will auto-generate on next build

### Package Manager

This project uses **pnpm** (version 8.15.5 as specified in package.json). Always use `pnpm` commands, not `npm` or `yarn`.

## Special Considerations

### Shared OpenAPI Schemas

The repository includes shared schema files in `docs/public/models/`:
- `event-1.0.0.yaml` - CloudEvents schema
- `problem-1.0.0.yaml` - RFC 9457 Problem Details schema
- `headers-1.0.0.yaml` - Common HTTP headers

These are referenced by the documentation and can be used as examples.

### Language Code Consistency

When working on documentation, maintain consistency between English and Traditional Chinese versions. Changes to document structure or frontmatter should be mirrored across both locales.

### VitePress Theme Customization

Custom theme components and styles are in `docs/.vitepress/theme/`:
- Custom Vue components for rendering YAML content
- RFC syntax highlighting
- Custom CSS for badges and layout
- Service Worker for PWA functionality

4. **Check conditional categories** based on the specific requirements of the API being designed
