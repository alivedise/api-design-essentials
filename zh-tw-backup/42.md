# [ADP-42] API 生命週期管理

## 指導原則

- 如果 API 的受眾大於 `COMPANY_INTERNAL`，必須使用以下標頭告訴客戶端 API 正在被棄用：Deprecation 和 Sunset。
- 即將退役的 API 首先進入棄用狀態，然後進入 Sunset 狀態，最後退役。
- 可以同時使用這兩個標頭。

### 棄用（Deprecation）

表示請求的 API 正在被棄用，將來可能不可用。

```jsx
Deprecation: @1688169599
```

* 應使用 RFC 草案中的結構化字段日期。結構化字段定義在 https://datatracker.ietf.org/doc/draft-ietf-httpbis-sfbis/ 的 3.3.7 節。
* 應在響應中提供鏈接，以向客戶端提供有關棄用的更多信息。參見 [HTTP Headers - Deprecation](127.md)。
### Sunset
表示請求的 API 將在指定的日期時間不可用。

```jsx
Sunset: Wed, 11 Nov 2026 11:11:11 GMT
```
* 應在 OpenAPI 文檔中標記 deprecated。
* 注意：Sunset 可以用於即將不可用的臨時資源。
