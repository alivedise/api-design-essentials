# [ADP-2] ADP 指南

## 指導原則

- 除非有需要創建新的類別，否則每個 ADP 應該屬於特定的現有 ADP 類別。
- 每個 ADP **應該** 包含代碼示例，以說明關鍵概念並提供實際使用場景。
- 每個 ADP **應該** 使用美式英語撰寫，以保持整個文檔的一致性。
- 每個 ADP **可以** 有繁體中文版本，以便非英語讀者閱讀。
- 每個 ADP **應該** 使用 Markdown 格式，便於編輯、閱讀和與各種工具的兼容性。
- 每個 ADP **不應該** 使用 H1 標題（`#`），因為標題已經佔用了 H1 級別。
- 每個 ADP **應該** 明確指定其類別來自預定義列表，以確保適當的組織和便於導航。
- 每個 ADP 應該在從草稿狀態轉換為發布狀態時附加變更日誌字段。

## ADP 類別

目前，我們將 ADP 分為以下類別：

- **ADP 本身**：與 ADP 框架及其組件相關的文檔和指南。
- **工程原則**：指導開發過程的核心工程原則和最佳實踐。
- **HTTP 基礎**：HTTP 的基本概念，包括方法、狀態碼和標頭。
- **REST 基礎**：設計 RESTful API 的原則和實踐，包括資源建模和 URI 設計。
- **HTTP 語義**：HTTP 語義的詳細解釋及其在 API 設計和實施中的應用。
- **JSON 原則**：結構化 JSON 數據的指南，包括命名約定和數據類型。
- **錯誤設計**：設計和處理 API 錯誤的最佳實踐，包括錯誤碼和響應格式。
- **事件設計**：設計事件驅動架構的原則，包括事件類型和有效負載結構。
- **命名約定**：API 中各種元素的標準化命名約定，如資源、參數和字段。
- **應用特定約定**：特定於應用領域或項目需求的自定義指南和約定。

這些類別可能會隨著我們不斷改進和擴展文檔框架而變更和重組。
