# [ADP-46] API 指南例外

每個團隊或業務單位在設計 API 時可能有特定的關注點或獨特的需求。ADP（API 設計原則）指南的作用是幫助和支持每個 API 設計者創建有效且一致的 API，而不是充當嚴格的規則執行者。ADP 指南作為一個有用的資源，提供最佳實踐和建議以促進設計過程，但不打算僵化地規定。

認識到在某些情況下，偏離已建立的約定可能有正當理由，特別是在目標受眾為 `COMPONENT_INTERNAL` 的 API 中。這些 API 旨在供特定組件或系統內部使用，因此可能有獨特的約束或需求，需對標準指南進行小幅調整。因此，允許在這些情況下合理地偏離約定是可以接受的，只要這些偏離不會損害 API 的整體完整性、可維護性或互操作性。這種靈活性旨在使團隊能夠滿足其特定需求，同時仍然遵守良好 API 設計的核心原則。
