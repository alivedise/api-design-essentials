# [ADP-3/D] ADP 編號與分類

## 指導原則

- 每個 ADP 必須有一個標識符，目前為編號。
- ADP 編號：0-1000 保留用於 HTTP API 設計的一般用途 ADP。
- 一般用途 ADP 可以定義其 ADP 編號範圍。
- ADP 編號：1000-2000 保留給 0-1000 範圍內的一般用途 ADP。
- ADP 編號：2000-2499 保留給 WebSocket API 設計。
- ADP 編號：2500-2999 保留給 GraphQL API 設計。
- ADP 編號：3000-3999 保留給異步 API 設計。
- ADP 編號：4000-4999 保留給非 HTTP 基礎的 API 設計。
- 應用程序/語言/組件/庫特定的 ADP 從 10000、20000、30000... 開始。
