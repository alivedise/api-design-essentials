# [ADP-47] API 認證與授權

## 指導原則

- 每個 API 端點應使用 OAuth2.0 或基於 JWT 的認證進行保護。
    - 為了確保 API 端點的安全訪問，實施 OAuth2.0 或 JWT（JSON Web Token）認證。這些方法提供了強大的安全機制，支持基於令牌的訪問，並能夠針對不同的使用情況啟用各種授權類型。
    - **示例：**
        - **OAuth2.0**：使用授權碼授權類型進行用戶認證。
            
            ```jsx
            POST /oauth/token HTTP/1.1
            Host: api.example.com
            Content-Type: application/x-www-form-urlencoded
            
            grant_type=authorization_code&code=AUTH_CODE&redirect_uri=REDIRECT_URI&client_id=CLIENT_ID&client_secret=CLIENT_SECRET
            ```
            
        - **JWT**：在每個 API 請求中驗證 JWT 令牌。
            
            ```jsx
            GET /protected/resource HTTP/1.1
            Host: api.example.com
            Authorization: Bearer <jwt_token>
            ```
            
- 不應使用基本認證。
    - 基本認證的安全性較低，因為它以易於解碼的格式傳輸用戶憑證。避免使用它來保護 API 端點。
    - 不要：
        
        ```jsx
        GET /protected/resource HTTP/1.1
        Host: api.example.com
        Authorization: Basic <base64_encoded_credentials>
        ```
        
- 應基於 HTTPS 構建。
    - 所有 API 通信必須使用 HTTPS 進行加密，以保護數據在傳輸過程中免受竊聽和中間人攻擊。確保正確配置和維護 SSL/TLS 證書。
    - 例外：內聯網設備可以不提供 HTTPS。
    - **示例：**（URL 應以 `https://` 開頭。）
        
        ```jsx
        GET /protected/resource HTTP/1.1
        Host: api.example.com
        ```
        
- **可以通過應用設計基於角色的授權控制（RBAC）。**
    - 實施基於角色的訪問控制，以有效管理用戶權限。定義角色並根據角色分配權限，以確保用戶僅能訪問他們被授權使用的資源。
    - **示例：**
        - 定義角色和權限：
            
            ```json
            {
              "roles": [
                {
                  "name": "admin",
                  "permissions": ["read", "write", "delete"]
                },
                {
                  "name": "user",
                  "permissions": ["read"]
                }
              ]
            }
            ```
            
        - 為用戶分配角色：
            
            ```json
            {
              "users": [
                {
                  "username": "alice",
                  "roles": ["admin"]
                },
                {
                  "username": "bob",
                  "roles": ["user"]
                }
              ]
            }
            ```
